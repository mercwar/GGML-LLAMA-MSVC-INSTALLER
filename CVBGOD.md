# 🧍 CVBGODS's GGML-LLAMA-MSVC-INSTALLER

I created this repository for anyone — humans or robots — who needs access to **precompiled LLM engine libraries**, including **llama.cpp**, **ggml**, and **GGUF support**, ready to use as a backend for your own AI assistant. These libraries are compiled with **MSVC**, and they include the `.a` and `.o` object files required by C programs and any language capable of loading portable object modules.

The headers are written in C, and some implementation files are C++, because that’s how the upstream llama.cpp project ships them. If you’re a C programmer, don’t worry — your LLM assistant (Copilot, Gemini, etc.) can generate any C++ wrappers you need without you having to modify anything manually.

This repository contains the exact libraries I use to load **FireGem** into my robot assistant. The original GCC build was converted into a full MSVC build, and I prefer it — so I recompiled everything from scratch and published the results here.

All you need to do is run the included batch installer. It will install the required tools, clone llama.cpp, compile the libraries, and prepare everything for your LLM assistant to take over. If you understand that GGUF models are tiny and require the GGML + LLAMA libraries to function, then you already understand why this repo exists: these libraries are the foundation for building your own assistant or studio.

A compact, MSVC‑driven build pipeline that:

- compiles them into **MSVC object files** (`.obj`, `.lib`)

## 🔩 How the Build System Works (Step‑by‑Step)

### 1. **Environment Setup**
The batch installer:

- installs **CMake**  
- installs **Ninja**  
- loads **MSVC BuildTools**  
- verifies Ninja is in PATH  

This ensures a clean, reproducible Windows build environment.

### 2. **Source Acquisition**
The installer checks for:

```
/llama.cpp
```

If missing, it clones the official repo:

```
git clone https://github.com/ggerganov/llama.cpp
```

This gives FireGem the exact upstream source tree.

### 3. **Build Directory Reset**
The script deletes any previous build:

```
rmdir /S /Q build
```

This prevents stale object files from contaminating the engine.

### 4. **CMake Configuration**
The installer configures llama.cpp using MSVC:

- `cl.exe` as C/C++ compiler  
- static runtime (`MultiThreaded`)  
- examples + tests enabled  
- Ninja as the generator  

This produces a clean MSVC‑compatible build graph.

### 5. **Compilation**
Ninja builds the entire engine:

```
cmake --build build --config Release
```

This produces:

- `ggml.obj`  
- `llama.obj`  
- `gguf.obj`  
- `common.obj`  
- `llama.lib`  
- `ggml.lib`  

These are the **core FireGem engine libs**.

### 6. **Verification**
If present, the installer runs:

```
llama-test.exe
```

This ensures the engine is functional.

**FireGem usage guide**  
Once your MSVC‑compiled llama.cpp + ggml + gguf backend is ready, FireGem becomes a lightweight, portable inference engine that your LLM assistant can drive. The guide below shows how to use the engine, how to load models, how to run inference, and how to build your own assistant on top of it.

---

## 🔥 Core idea  
FireGem is not a “program.”  
It is a **backend** — a compiled set of llama.cpp libraries — that your LLM assistant (Copilot, Gemini, Claude, etc.) can use to generate the rest of your AI assistant.

You provide:

- the **compiled engine**  
- a **GGUF model**  
- a **main.c / main.cpp** file generated by your LLM  

FireGem provides:

- GGML tensor math  
- LLAMA inference logic  
- GGUF model loading  
- MSVC‑compatible object files  

Your LLM provides:

- the assistant logic  
- the chat loop  
- the sampling strategy  
- the UI or robot interface  

---

## ⚙️ 1. Include the engine headers  
Your project must include the engine’s header directories:

- `engine/include/`  
- `engine/ggml/`  
- `engine/src/`  

In MSVC:

```
Project Properties → C/C++ → Additional Include Directories
```

Add:

```
engine/include
engine/src
engine/ggml
```

---

## 📦 2. Link the compiled libraries  
Your build must link the MSVC‑compiled `.lib` files:

- `llama.lib`  
- `ggml.lib`  
- `common.lib` (if present)  

In MSVC:

```
Project Properties → Linker → Additional Dependencies
```

Add:

```
llama.lib
ggml.lib
common.lib
```

---

## 📁 3. Place your GGUF model  
Put your model anywhere you want, for example:

```
models/myassistant.gguf
```

Your LLM assistant will generate code that loads it.

---

## 🧠 4. Load the model (LLM‑generated code)  
Your LLM assistant will generate something like:

```cpp
llama_model_params mparams = llama_model_default_params();
llama_model* model = llama_load_model_from_file("models/myassistant.gguf", mparams);

llama_context_params cparams = llama_context_default_params();
llama_context* ctx = llama_new_context(model, cparams);
```

This is the core of FireGem:  
**load GGUF → create context → run inference.**

---

## 💬 5. Tokenize user input  
Your LLM assistant will generate:

```cpp
std::vector<llama_token> tokens = llama_tokenize(ctx, user_input, true);
```

---

## 🔁 6. Run inference  
Your LLM assistant will generate:

```cpp
llama_eval(ctx, tokens.data(), tokens.size());
```

Then it will generate a loop to read tokens:

```cpp
llama_token next = llama_sample_token(ctx);
std::cout << llama_token_to_str(ctx, next);
```

---

## 🔄 7. Build a chat loop  
Your LLM assistant will generate:

```cpp
while (true) {
    std::string user;
    std::getline(std::cin, user);

    if (user == "exit") break;

    // tokenize → eval → print tokens
}
```

This becomes your **AI assistant**.
